31. Τι συμβαίνει κατά την ανταλλαγή μνήμης;

ΑΠΑΝΤΗΣΗ

Αυτή η τεχνική χρησιμοποιείται όταν η φυσική μνήμη είναι περιορισμένη και το σύστημα χρειάζεται να διαχειριστεί περισσότερες διεργασίες από ό,τι η διαθέσιμη μνήμη μπορεί να υποστηρίξει ταυτόχρονα. Αν κάποια διεργασία χρειάζεται περισσότερη μνήμη από αυτή που διατίθεται, τα δεδομένα της μεταφέρονται στον δίσκο και επιστρέφουν στη μνήμη όταν απαιτείται εκ νέου.

Διαδικασία ανταλλαγής:

  1. Αναγνώριση υπερφόρτωσης μνήμης: Όταν η φυσική μνήμη δεν αρκεί για να φιλοξενήσει όλες τις διεργασίες.
  2. Μεταφορά δεδομένων στον δίσκο: Τα δεδομένα που δεν είναι άμεσα απαραίτητα μεταφέρονται στον δίσκο.
  3. Ανάκτηση δεδομένων: Όταν μια διεργασία χρειάζεται τα δεδομένα που βρίσκονται στο δίσκο, αυτά επιστρέφουν στη μνήμη και αντικαθιστούν τα δεδομένα μιας άλλης διεργασίας που μπορεί να αποθηκευτεί ξανά στον δίσκο.

36. Τι είναι ημίδιπλη (ταυτόχρονα αμφίδρομη - Full Duplex) επικοινωνία και τι η ημίδιπλη (αμφίφρομη μη ταυτόχρονη - Half Duplex) επικοινωνία. Να αναφέρετε τις διαφορές μεταξύ τους.

ΑΠΑΝΤΗΣΗ

  • Ημι-διπλή επικοινωνία (Half Duplex): Επιτρέπει τη μετάδοση δεδομένων σε μία κατεύθυνση τη φορά. Κάθε συσκευή μπορεί είτε να στέλνει είτε να λαμβάνει δεδομένα, αλλά όχι ταυτόχρονα. Παράδειγμα: Ραδιόφωνο CB.

  • Διπλή επικοινωνία (Full Duplex): Επιτρέπει τη ταυτόχρονη μετάδοση και λήψη δεδομένων και στις δύο κατευθύνσεις. Παράδειγμα: Τηλεφωνική συνομιλία.

Διαφορές:

  • Κατεύθυνση μετάδοσης: Στην ημι-διπλή, η μετάδοση είναι είτε προς μία είτε προς την άλλη κατεύθυνση, ενώ στη διπλή είναι ταυτόχρονη και στις δύο κατευθύνσεις.
  • Χρόνος μετάδοσης: Στην ημι-διπλή, ο χρόνος μετάδοσης μοιράζεται μεταξύ των συσκευών, ενώ στη διπλή, η μετάδοση είναι συνεχής και αμφίδρομη.

Το ATM (Asynchronous Transfer Mode) είναι ένα πρωτόκολλο δικτύου που χρησιμοποιείται για τη μετάδοση δεδομένων, φωνής και βίντεο μέσω ενός κοινού δικτύου. Η δομή του βασίζεται σε μικρά, σταθερού μεγέθους πακέτα δεδομένων, γνωστά ως “cells”, που επιτρέπουν την αποτελεσματική και αξιόπιστη μετάδοση.

Οι κύριες κατηγορίες υπηρεσιών που προσφέρει το ATM είναι:

  • CBR (Constant Bit Rate): Υπηρεσία με σταθερό ρυθμό μετάδοσης, κατάλληλη για εφαρμογές όπως η τηλεφωνία.
  • VBR (Variable Bit Rate): Υπηρεσία με μεταβλητό ρυθμό μετάδοσης, κατάλληλη για εφαρμογές πολυμέσων.
  • ABR (Available Bit Rate): Υπηρεσία που προσαρμόζει τον ρυθμό μετάδοσης με βάση τη διαθεσιμότητα του δικτύου.
  • UBR (Unspecified Bit Rate): Υπηρεσία χωρίς εγγυημένο ρυθμό μετάδοσης, κατάλληλη για εφαρμογές που μπορούν να ανεχτούν καθυστερήσεις.

    • Αποτέλεσμα: Μείωση του θορύβου και των ανεπιθύμητων διαφορών χρωμάτων, δημιουργώντας μια πιο ομαλή εμφάνιση.

  4. Φίλτρα Μετατροπής (Transformation Filters):

43. Περιγράψτε ποιο είναι το περιεχόμενο σ’ ένα αρχείο διανυσματικών γραφικών.

44. Αναφέρετε πλεονεκτήματα και μειονεκτήματα των διανυσματικών γραφικών.

Πλεονεκτήματα:

  • Ανεξαρ1010τησία Ανάλυσης (Resolution Independence): Τα διανυσματικά γραφικά μπορούν να μεγεθύνονται ή να σμικρύνονται χωρίς απώλεια ποιότητας, καθώς βασίζονται σε μαθηματικές περιγραφές αντί για πίνακες εικονοστοιχείων.

45. Ποιες είναι οι βασικές διαφορές μεταξύ bitmap graphics και vector graphics;

ΑΠΑΝΤΗΣΗ

  1. Δομή Απεικόνισης:

    • Bitmap (Raster) Γραφικά: Αποτελούνται από ένα πλέγμα (grid) μικρών στοιχείων, γνωστών ως pixels, όπου κάθε pixel περιέχει πληροφορίες για το χρώμα και την έντασή του.
    • Vector Γραφικά: Χρησιμοποιούν μαθηματικές εξισώσεις για να περιγράψουν γεωμετρικά σχήματα όπως γραμμές, καμπύλες και πολύγωνα, βασισμένα σε σημεία (κόμβους) και διαδρομές (paths).

  2. Κλιμάκωση και Ανάλυση:

    • Bitmap Γραφικά: Έχουν σταθερή ανάλυση και η ποιότητά τους μειώνεται όταν μεγεθύνονται, εμφανίζοντας θολούρα ή pixelation.
    • Vector Γραφικά: Είναι ανεξάρτητα από την ανάλυση, επιτρέποντας απεριόριστη κλιμάκωση χωρίς απώλεια ποιότητας.

  3. Μέγεθος Αρχείου:

    • Bitmap Γραφικά: Μπορεί να έχουν μεγαλύτερο μέγεθος αρχείου, ειδικά σε εικόνες υψηλής ανάλυσης, καθώς αποθηκεύουν δεδομένα για κάθε pixel.
    • Vector Γραφικά: Συνήθως έχουν μικρότερο μέγεθος αρχείου, καθώς αποθηκεύουν μόνο τις μαθηματικές παραμέτρους των σχημάτων.

  4. Επεξεργασία και Ευελιξία:

    • Bitmap Γραφικά: Η επεξεργασία μπορεί να είναι περιορισμένη, καθώς η τροποποίηση μεμονωμένων pixels μπορεί να είναι χρονοβόρα και να απαιτεί εξειδικευμένα εργαλεία.
    • Vector Γραφικά: Επιτρέπουν εύκολη επεξεργασία, όπως αλλαγή χρώματος, μεγέθους ή σχήματος, χωρίς να επηρεάζεται η ποιότητα της εικόνας.

  5. Κατάλληλες Εφαρμογές:

    • Bitmap Γραφικά: Ιδανικά για φωτογραφίες και εικόνες με περίπλοκες χρωματικές λεπτομέρειες.
    • Vector Γραφικά: Κατάλληλα για λογότυπα, εικονογραφήσεις, διαγράμματα και γραφικά που απαιτούν κλιμάκωση χωρίς απώλεια ποιότητας.

46. Περιγράψτε τις διαφορές μεταξύ των προτύπων TIFF, JPEG, GIF. Που χρησιμοποιείται το καθένα;

ΑΠΑΝΤΗΣΗ

1. TIFF (Tagged Image File Format)

  • Χαρακτηριστικά:

    • Χρησιμοποιεί μη καταστρεπτική συμπίεση ή καθόλου συμπίεση, διατηρώντας την ποιότητα της εικόνας.
    • Υποστηρίζει πολλαπλές σελίδες (αρχεία με πολλές εικόνες).
    • Υποστηρίζει αμφίδρομη συμπίεση (Lossless και Lossy).
    • Υποστηρίζει διαφάνεια και μεγάλες αναλύσεις.
    • Συχνά χρησιμοποιείται για εκτυπωτικά έργα ή εικόνες υψηλής ανάλυσης, όπου η ποιότητα είναι κρίσιμη.

  • Χρήση:

    • Χρησιμοποιείται κυρίως σε εφαρμογές επαγγελματικής εκτύπωσης και σε εικόνες που απαιτούν υψηλή ποιότητα χωρίς απώλεια δεδομένων (π.χ., σκαναρίσματα, φωτογραφίες επαγγελματικού επιπέδου).

2. JPEG (Joint Photographic Experts Group)

  • Χαρακτηριστικά:

    • Χρησιμοποιεί καταστρεπτική συμπίεση (Lossy compression), πράγμα που σημαίνει ότι χάνεται μέρος των δεδομένων της εικόνας για να μειωθεί το μέγεθος του αρχείου.
    • Δεν υποστηρίζει διαφάνεια.
    • Πολύ ευρέως χρησιμοποιούμενο για εικόνες σε ιστοσελίδες, φωτογραφίες και γενικά αρχεία εικόνας με χρωματική πληροφορία.

  • Χρήση:

    • Ιδανικό για φωτογραφίες και εικόνες που δεν απαιτούν επεξεργασία μετά την αποθήκευση, καθώς το μέγεθος του αρχείου είναι μικρό και η ποιότητα είναι επαρκής για τη χρήση στο web και στις ψηφιακές φωτογραφίες.

3. GIF (Graphics Interchange Format)

  • Χαρακτηριστικά:

    • Χρησιμοποιεί συμπίεση χωρίς απώλειες (Lossless compression).
    • Υποστηρίζει μόνο 256 χρώματα.
    • Μπορεί να υποστηρίξει κινούμενες εικόνες (GIF animation), δημιουργώντας σειρές εικόνων που εναλλάσσονται σε μια κίνηση.
    • Υποστηρίζει διαφάνεια (αλλά μόνο 1 χρώμα μπορεί να είναι διαφανές).

  • Χρήση:

    • Χρησιμοποιείται κυρίως για κινούμενες εικόνες και εικόνες χαμηλής ανάλυσης στο διαδίκτυο.
    • Ιδανικό για απλές εικόνες, εικονίδια και μικρές κινούμενες εικόνες σε ιστοσελίδες.

Σύνοψη Χρηστικών Πεδίων:

  • TIFF: Χρησιμοποιείται σε εφαρμογές που απαιτούν πολύ υψηλή ποιότητα εικόνας, όπως σκανάρισμα, εκτύπωση και αρχείο φωτογραφιών υψηλής ανάλυσης.
  • JPEG: Ιδανικό για φωτογραφίες και εικόνες με πολλές αποχρώσεις, όπου η συμπίεση είναι σημαντική για τη μείωση του μεγέθους του αρχείου.
  • GIF: Χρησιμοποιείται για κινούμενα σχέδια ή εικόνες χαμηλής ανάλυσης με περιορισμένα χρώματα και υποστήριξη διαφάνειας.

48. Εξηγήστε την τεχνική Antialiasing και Alpha channel που χρησιμοποιείται σε εφαρμογές επεξεργασίας εικόνας και φωτορεαλιστικής απεικόνισης αντικειμένων.

ΑΠΑΝΤΗΣΗ

Antialiasing:

  • Η τεχνική Antialiasing χρησιμοποιείται για την εξομάλυνση των “σκαλωσιών” (jaggies) που εμφανίζονται στις διαγώνιες ή καμπύλες γραμμές σε ψηφιακές εικόνες.
  • Αυτό επιτυγχάνεται με την προσθήκη ενδιάμεσων χρωμάτων μεταξύ των ακμών των αντικειμένων και του φόντου, δημιουργώντας την ψευδαίσθηση ομαλότητας.

Alpha Channel:

  • Το Alpha Channel είναι ένα επιπλέον κανάλι σε μια εικόνα που καθορίζει την διαφάνεια κάθε pixel.
  • Επιτρέπει την ανάμειξη μιας εικόνας με το φόντο της, δημιουργώντας εφέ όπως η σκίαση, η διάχυση του φωτός και άλλες φωτορεαλιστικές απεικονίσεις.
  • Στις εφαρμογές επεξεργασίας εικόνας, το alpha channel χρησιμοποιείται για να δημιουργήσει σύνθετες εικόνες με διαφάνειες, επιτρέποντας την τοποθέτηση αντικειμένων πάνω σε διάφορα υπόβαθρα με φυσικό τρόπο.

132. Δημιουργείστε σε JAVA απλό κινούμενο αντικείμενο για το παιχνίδι σας (πχ μπαλάκι του τένις),ορίζοντας πρώ0τα την αρχική του θέση με x και y και κατόπιν τις επόμενες ανάλογα με την κίνηση που θέλετε να πάρει.

ΑΠΑΝΤΗΣΗ 
import javax.swing.*;
import java.awt.*;

public class MovingBall extends JPanel {
    private int x = 100, y = 100; // Αρχική θέση του μπα239λακιού
    private int dx = 1, dy = 1; // Ταχύτητα κίνησης

    public MovingBall() {
        Timer timer = new Timer(10, e -> moveBall());
        timer.start();
    }

    public void moveBall() {
        // Κίνηση του μπαλακιού
        x += dx;
        y += dy;
        if (x >= getWidth() || x <= 0) dx = -dx; // Αντικατοπτρισμός στις άκρες
        if (y >= getHeight() || y <= 0) dy = -dy; // Αντικατοπτρισμός στις άκρες
        repaint();
    }

    @Override
    protected void paintComponent(Graphics g) {
        super.paintComponent(g);
        g.setColor(Color.RED);
        g.fillOval(x, y, 30, 30); // Σχεδίαση του μπαλακιού
    }

    public static void main(String[] args) {
        JFrame frame = new JFrame("Moving Ball");
        MovingBall ball = new MovingBall();
        frame.setSize(400, 400);
        frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        frame.add(ball);
        frame.setVisible(true);
    }
}

  • Διαφορά μεταξύ καρέ: Κωδικοποιεί μόνο τις αλλαγές μεταξύ των καρέ, όχι όλα τα καρέ, μειώνοντας έτσι τον όγκο των δεδομένων.

  • Χώρος και χρονική συμπίεση: Συνδυάζει την συμπίεση τόσο του χώρου όσο και του χρόνου για να εξασφαλίσει καλύτερη απόδοση χωρίς να επηρεάζει την ποιότητα.

157. Υπολογίστε το μέγεθος σε Kbytes από τον αποθηκευτικό χώ0ρο ενός σκληρού δίσκου που δεσμεύεται κατά τη σύλληψη ασυμπίεστου ψηφιακού video χωρίς ήχο από βιντεοσυσκευή με σύνθετο βιντεοσήμα PAL, διάρκειας 2 λεπτών, με διαστάσεις 320 x 240 εικονοστοιχείων και βάθος χρώματος 16bit.

ΑΠΑΝΤΗΣΗ

Για να υπολογίσουμε το μέγεθος, χρησιμοποιούμε την παρακάτω φόρμουλα:

\[ \text{Μέγεθος αρχείου} = \text{Διάσταση εικόνας} \times \text{Βάθος χρώματος} \times \text{Αριθμός καρέ ανά δευτερόλεπτο} \times \text{Χρόνος} \]

  • Διαστάσεις εικόνας: 320 x 240
  • Βάθος χρώματος: 16 bit = 2 bytes
  • Ρυθμός καρέ: 25 καρέ ανά δευτερόλεπτο (για PAL)
  • Διάρκεια: 2 λεπτά = 120 δευτερόλεπτα

\[ \text{Μέγεθος αρχείου} = 320 \times 240 \times 2 \times 25 \times 120 = 23,040,000 \text{ bytes} = 22.03 \text{ MB} \]

Για να μετατρέψουμε σε Kbytes:

\[ 22.03 \text{ MB} = 22,030 \text{ KB} \]

Άρα, το μέγεθος του αρχείου είναι 22,030 KB.

function DistanceTwoPoints(x1:Number, y1:Number, x2:Number, y2:Number):Number {
    var dx:Number = x2 - x1; // Υπολογισμός διαφοράς x
    var dy:Number = y2 - y1; // Υπολογισμός διαφοράς y

if (mc_1.x > 800) {
    mc_1.x = 0;
} else if (mc_1.x < 0) {
    mc_1.x = 800;

// Collision detection
if (mc_1.hitTestObject(mc_2)) {
    // Αν υπάρχει σύγκρουση, αναστρέφουμε την κατεύθυνση του mc_1
    speed = -speed;
}
